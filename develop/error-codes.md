# <a name="partner-center-rest-error-codes"></a>REST-foutcodes voor Partnercentrum

**Van toepassing op:**

- Partnercentrum
- Partnercentrum beheerd door 21Vianet
- Partnercentrum voor Microsoft Cloud Duitsland
- Partnercentrum voor Microsoft Cloud for US Government

De REST Api's van het Partner Center retour neren een JSON-object dat een status code bevat. Deze code geeft aan of uw aanvraag is geslaagd of waarom deze is mislukt.

## <a name="success-responses"></a>Geslaagde antwoorden

Een **2xx** -status code geeft aan dat de aanvraag van de client is ontvangen, begrepen en geaccepteerd.

## <a name="http-status-codes"></a>HTTP-statuscode

De volgende **4xx** -en **5xx** -status codes geven aan dat er een fout is opgetreden:

- 400: ongeldige aanvraag
- 401: niet geautoriseerd
- 403: verboden
- 404: niet gevonden
- 405: methode niet toegestaan
- 406: niet toegestaan
- 409: conflict, fout code
- 412: de voor waarde is mislukt
- 429: te veel aanvragen
- 500: interne server fout
- 501: niet geïmplementeerd
- 502: ongeldige gateway
- 503: de service is niet beschikbaar
- 504: time-out van Gateway

## <a name="error-responses"></a>Foutreactiess

Elk antwoord met een **4xx** -of **5xx** -status code bevat een fout bericht met aanvullende informatie over de fout voorwaarde (n) die zijn opgetreden.

In het volgende tabel-en code voorbeeld wordt het schema van een fout bericht beschreven:

| Naam        | Type   | Beschrijving                                                                                                                                            |
|-------------|--------|--------------------------------------------------------------------------------------------------------------------------------------------------------|
| code        | tekenreeks | Altijd geretourneerd. Hiermee wordt het soort fout aangegeven dat is opgetreden. Niet-null.                                                                                  |
| beschrijving | tekenreeks | Altijd geretourneerd. Hierin wordt de fout in details beschreven en vindt u meer informatie over fout opsporing. Niet-null en niet-leeg. De maximale lengte is 1024 tekens. |
| gegevens        | matrix  | Wordt alleen geretourneerd voor bepaalde fout typen. Een lijst met fout objecten.                                                                                           |
| source      | tekenreeks | Altijd geretourneerd. De bron van de fout.                                                                                                              |

```json
{
  "code": <string>,
  "description": <string>,
  "data": [

  ],
  "source": <string>
}
WWW-Authenticate: OAuth realm=urn:cpsvc:cpid:{some cid}
```
